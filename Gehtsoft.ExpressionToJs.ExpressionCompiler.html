Brief

Converts a C# lambda expression into an equivalent JavaScript expression string.

Details

This is the entry point of the library. Construct it around a lambda - typically a boolean form-validation predicate - and read ExpressionCompiler.JavaScriptExpression to get JavaScript that computes the same result in the browser. The point is to keep one source of truth: the lambda runs server-side as a compiled delegate, and its emitted twin runs client-side, so a rule cannot drift between the two ends.

Configure framing and translation before reading the expression: set ExpressionCompiler.DateMode for date semantics, and register custom translations through ExpressionCompiler.Methods , ExpressionCompiler.Constants , and ExpressionCompiler.Members . For deeper control - notably how a parameter renders on the client - derive a subclass and override the protected virtual emit methods. Whatever the emitted code calls (the jsv_* helpers) must be present at evaluation time; see ExpressionToJsStubAccessor.GetJsIncludesAsString .
