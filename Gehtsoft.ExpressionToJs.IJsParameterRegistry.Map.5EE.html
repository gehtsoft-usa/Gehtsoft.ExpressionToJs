public method IJsParameterRegistry.Map

Brief

Bind the matching rule parameters with custom rendering.

Declaration
C#
Gehtsoft.ExpressionToJs.IJsParameterRegistry  Map (Func<Type,bool> matches, Func<ParameterExpression,string> parameter, Func<Expression,ParameterExpression,string> parameterAccess)

Parameters
matches

Tells whether a rule parameter of the given type should bind.
parameter

Renders the whole model - the bare parameter. The gives its type (and name) - for example p => "value", or p => "reference('" + p.Type.Name + "')".
parameterAccess

Renders a field access, given the member/index chain and the root parameter it is rooted in. Use ExpressionCompiler.ParameterAccessPath to turn the chain into a dotted path - for example (e, p) => "value." + ExpressionCompiler.ParameterAccessPath(e) turns m.Address.PostalCode into value.Address.PostalCode.

Details

Use this when the host hook is not the default reference() shape - for example a validated value bound to an ambient value, or a differently named lookup. As with

IJsParameterRegistry.MapReference , only the rule's own parameters are matched.
