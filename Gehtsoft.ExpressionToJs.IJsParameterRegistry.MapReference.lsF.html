public method IJsParameterRegistry.MapReference

Brief

Bind the matching rule parameters to the built-in reference() convention.

Declaration
C#
Gehtsoft.ExpressionToJs.IJsParameterRegistry  MapReference (Func<Type,bool> matches)

Parameters
matches

Tells whether a rule parameter of the given type should bind to a reference.

Details

The bare parameter emits as reference() and a member chain rooted in it emits as reference('Path.To.Field') (array indexing becomes jsv_index(...)) - the page supplies a reference(path) function that returns the field's current value. This is the common case; use IJsParameterRegistry.Map for any other shape.

The match runs on the parameter's type, but usually the exact type does not matter - a predicate such as t => !t.IsPrimitive & t != typeof(string) (anything that looks like a model) is typical. Only the rule's own (top-level) parameters are matched; parameters introduced by a nested LINQ lambda always keep their own name.
