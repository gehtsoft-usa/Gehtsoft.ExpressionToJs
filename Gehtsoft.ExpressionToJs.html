Namespace Gehtsoft.ExpressionToJs

Brief

The common namespace for all of the library's classes.

Details

Entry points

Type

Purpose

ExpressionCompiler

The entry point. Wrap a lambda, optionally configure it, then read JavaScriptExpression to get the emitted code.

ExpressionToJsStubAccessor

Hands you the JavaScript runtime (stub.js) that the emitted code calls into. Load it into the page once.

Functions

Validation helpers (age math, Luhn check, lenient coercions, null tests) safe to call inside a rule, because each one has a matching JavaScript implementation.

DateTimeMode

Selects how dates are framed in the emitted code so both ends agree on the calendar.

Extending

The compiler can be taught to translate methods, properties and constants on types the built-ins do not cover. For when and how, see Extending the translator for your own types.

Classes

ExpressionCompiler

Converts a C# lambda expression into an equivalent JavaScript expression string.

ExpressionToJsStubAccessor

Gives you the JavaScript runtime that every compiled expression depends on.

Functions

Validation helpers you call from inside the lambdas you compile to JavaScript.

IConstantTranslator

Translates a constant value into a JavaScript literal.

IExpressionEmitContext

The building blocks a method-call translator needs to emit JavaScript.

IJsConstantRegistry

Additive registry for custom constant types (Guid, enums, value structs, ...).

IJsMemberRegistry

Additive registry for custom member/property access.

IJsMethodRegistry

Additive extension surface for mapping method calls to JavaScript.

IJsParameterRegistry

Registry for binding a parameter type to a host-side lookup instead of emitting it verbatim.

IMemberTranslator

Translates a member/property access into JavaScript.

IMethodCallTranslator

Translates a specific family of method calls into JavaScript.

Enumerations

DateTimeMode

How DateTime values are framed in the emitted JavaScript.
