Dates and time zones

Brief

Pick a date frame with DateTimeMode so the server and the browser agree on the calendar.

Details

JavaScript Date objects carry a time zone; C# DateTime values are ambiguous about one. To keep the two ends in agreement, the compiler emits every calendar construct in a frame you choose: DateTimeMode.

Choosing the frame

Mode

Emits

DateTimeMode.Local (default)

Local-time constructs - new Date(y, m, d, ...) and getFullYear() / getMonth() / ...

DateTimeMode.Utc

UTC constructs - new Date(Date.UTC(y, m, d, ...)) and getUTCFullYear() / getUTCMonth() / ...

Set the mode through the DateMode property or the ExpressionCompiler(lambda, mode) constructor. It must be set before you read JavaScriptExpression, because the result is computed once and cached.

var compiler = new ExpressionCompiler(rule, DateTimeMode.Utc);
// or: compiler.DateMode = DateTimeMode.Utc;  (before reading JavaScriptExpression)
string js = compiler.JavaScriptExpression;

The contract

Bind JS-side dates in the same frame you select here. A mismatch does not error - it silently shifts every calendar-component comparison by the client's UTC offset, so a rule can quietly pass or fail by up to a day.

The mode only governs constructs that read or build calendar components: date constants, the Year/Month/Day/Hour/... reads, and the calendar-based MonthsSince / YearsSince helpers (which receive a UTC flag).

What is frame-independent

Operations that work on the absolute instant (the epoch millisecond) are the same in both frames, so you never have to think about the mode for them:

  • Adding or subtracting a TimeSpan, and subtracting two dates.

  • AddDays / AddHours / AddMinutes / AddSeconds.

  • The TimeSpan.Total* properties and DaysSince.

The clock stays live

DateTime.Now, DateTime.Today and DateTime.UtcNow are not folded into a compile-time constant. They emit live JavaScript - Now/UtcNow become new Date(), and Today becomes jsv_today(...) (mode-aware midnight) - so the rule is evaluated against the clock when the user submits, exactly as the server would. Chains such as DateTime.Now.AddDays(30) stay dynamic too.
