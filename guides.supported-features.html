Supported expression features

Brief

The C# operators, methods and constructs you can use in a rule, and the ones you cannot.

Details

A rule is an expression tree, and the compiler translates the constructs listed here. Anything not listed throws at compile time (when you read JavaScriptExpression), so an unsupported construct fails loudly on the server rather than silently misbehaving in the browser.

Two principles run through everything below. First, C# semantics win: the emitted jsv_* helpers reproduce C# behaviour, not JavaScript's - integer division truncates toward zero, int arithmetic wraps on overflow, Math.Round uses banker's rounding, and == is typed and strict (no "10" == 10); see Unit-testing your expressions. Second, constant subtrees are folded: a subtree with no free parameter and no ambient Now/Today/UtcNow is evaluated at compile time and emitted as a literal.

Operators

Category

Supported

Arithmetic

+ - * / %, unary -, and power. Integer division truncates and int results wrap, matching C#.

Comparison

< <= > >= == !=. Equality is strict and typed; two numbers compare with a small tolerance so 0.1 + 0.2 == 0.3 holds.

Logical

&& || !, and logical XOR (^ on bool).

Bitwise

& | ^ on integers (wrapped to 32-bit).

Other

Conditional ?:, null-coalescing ??, array indexing a[i], and array/collection length.

Methods, members and helpers

Area

Supported

Math (System.Math)

Round (banker's, including the Round(x, digits) overload), Floor, Ceiling, Truncate, Abs, Sign, Sqrt, Pow, Log, Exp, Min, Max, the trigonometric functions, and Math.PI.

Strings

Trim/TrimStart/TrimEnd, ToUpper/ToLower (and the Invariant variants), StartsWith/EndsWith (including the StringComparison overload), Contains, IndexOf/LastIndexOf, Replace, PadLeft/PadRight (with optional pad char), Length, the indexer, and ToString.

Static string tests

string.IsNullOrEmpty, string.IsNullOrWhiteSpace (true Unicode whitespace, matching C#).

Characters

char.IsUpper, IsLower, IsDigit, IsLetter, IsLetterOrDigit, IsPunctuation, IsControl, IsWhiteSpace.

Regex

Regex.IsMatch(input, pattern). The pattern is escaped into a JS regex literal.

Collections (LINQ)

Any, All, Count, First, Last, FirstOrDefault/LastOrDefault (with a predicate), Where, Select, Sum, Min, Max, Distinct, Contains. They return real JS values, so they chain.

Dates and times

Component reads Year/Month/Day/Hour/Minute/Second/DayOfWeek (framed by DateTimeMode); AddDays/AddHours/AddMinutes/AddSeconds; adding/subtracting a TimeSpan; subtracting two dates; the TimeSpan.Total* properties; and the dynamic clock Now/Today/UtcNow. See Dates and time zones.

Parity helpers (Functions)

Checks with no direct BCL-to-JS equivalent, shipped as matched pairs (a Functions method and an identical jsv_* twin): DaysSince, MonthsSince, YearsSince, IsCreditCardNumberCorrect (Luhn), ToBool, ToInt, Fractional, IsNull, IsNotNull, IsNullOrEmpty, IsNotNullOrEmpty.

Enums

Emitted as the underlying numeric value (flags enums combine), so client-side enum fields must carry the numeric value. To emit an enum by name instead, register a constant translator - see Extending the translator for your own types.

Not supported

These throw at compile time, so they fail loudly on the server rather than misbehaving in the browser:

Construct

Why / alternative

String + concat, string.Format, string.Split

Locale and format correctness on the JS side is out of scope. (+ on strings compiles to a string.Concat call, not an Add.)

DateTime.AddMonths/AddYears, DateTime.Parse, TimeSpan.FromX, DateTimeOffset

Not translated; use the supported date arithmetic, or a custom translator.

Bit shifts << >>, int.TryParse/double.TryParse, LINQ Average, predicate-less FirstOrDefault/LastOrDefault

Not yet translated.

long beyond 2^53

Loses precision in JavaScript and is not made safe.

Statement-bodied lambdas

Use a single expression.

The not-supported list reflects the current built-ins; most gaps can be closed for your own code with a custom translator - see Extending the translator for your own types.
