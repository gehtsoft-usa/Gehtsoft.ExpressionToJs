Gehtsoft.ExpressionToJs

Brief

Compile a C# validation rule once and run it on both the server and the browser.

Details

One rule, two runtimes, no drift.

Form validation almost always has to exist twice: once on the server, where it is authoritative and cannot be bypassed, and once in the browser, where it gives the user instant feedback without a round trip. Written by hand, those two copies drift apart - a rule tightened on the server is forgotten on the client, an edge case fixed in JavaScript never makes it back into C#, and the two ends quietly disagree about what "valid" means.

Gehtsoft.ExpressionToJs removes the second copy. You write the rule once, as an ordinary C# lambda - typically a boolean predicate over your model. The server runs that lambda directly, compiled to a normal delegate. The same lambda is translated into an equivalent JavaScript expression that runs in the browser. There is a single source of truth, so the client and server cannot disagree: change the C# predicate and both ends change together.

Because the emitted code is a faithful translation - same operators, same string and date handling, same null and numeric semantics - a value that passes (or fails) on one side passes (or fails) the same way on the other.

The API at a glance

Type

Purpose

ExpressionCompiler

The entry point: wrap a lambda, read the emitted JavaScript.

Functions

Validation helpers (age, Luhn check, lenient coercions, null tests) safe to call inside a rule, because each one has a matching JavaScript implementation.

ExpressionToJsStubAccessor

The JavaScript runtime the emitted expressions depend on; load it into the page once.

DateTimeMode

Selects how dates are framed (local time or UTC) so both ends agree on the calendar.
